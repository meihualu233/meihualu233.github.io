Windows下Hexo博客迁移到另一个系统

日常
  • 博客迁移

周末的时候给电脑配了个固态硬盘，然后把系统放到了固态上，所以原来系统上的一些东西就都乱了。原来搭好的这个博客也就需要进行一次迁移。

将博客从原来的系统迁移到新的系统

1.从Git官网上下载Git，然后需要生成新的密钥。具体做法就是，

生成的密钥文件默认在 C:|用户|系统用户|.ssh中，将其中id_rsa.pub的公钥导入到git上即可。

2.下载安装Node.js
3.打开git bash 安装Hexo

4.下面就是重点了（特别感谢知乎上的skycrown，我百度到的方法）
拷贝原来博客目录下的 _config.yml ,theme/ ,source/,scaffolds/,package.json到新的目录，其他的内容就可以不要了。然后在拷贝的目录里,执行。

注：可能是我下载git bash有问题或者我的环境有问题，我直接在新的博客路径中右键打开git bash，npm 命令会报错 （bash: npm: command not found）
。在桌面邮件打开的git bash之后cd到博客路径下npm install就可以直接使用，如果有人出现相同的问题可以参考下。

注注：在输入完npm install 会出现这样的警告 this package is no longer maintained。不用慌张等一会就好了。

5.装好后，直接hexo g | hexo d 就好了，和以前一样操作就好了

6.但是发现了一个问题。我原来的博客置顶的内容错误了。原因是Hexo只支持按照时间排序，所以我以前修改过..\node_modules\hexo-generator-index\lib\generator.js中的内容，再次修改一下置顶功能就好了。
附一个我的generator.js增加置顶的配置.

END.
